# Life Dashboard

這是我自己在用的個人財務與生活儀表板。一開始只是想記帳，後來把資產、股票、習慣、倒數日、想買的東西全都塞進同一個頁面，就變成現在這樣了。

整套東西沒有自己的伺服器、也沒有正規資料庫。前端就是一支 `index.html`（HTML、CSS、JS 全寫在同一個檔案裡），資料全部放在一份 Google Sheet，中間靠一支 Google Apps Script（`backend.gs`）當 API。會這樣搞純粹是因為免費、不用養機器，而且資料本身就是一張試算表 —— 哪天程式壞了，我至少還能直接打開 Google Sheet 看跟改。

## 由三塊組成

- **前端 `index.html`**：放在 GitHub Pages 上。畫面分四頁：首頁、記帳、資產、更多。圖表用 Chart.js（CDN 載入）。
- **後端 `backend.gs`**：部署成 Google Apps Script 的 Web App。`doGet` 負責讀某個分頁回傳 JSON，`doPost` 負責寫入跟刪除。寫入時會用 `LockService` 上鎖，避免兩個請求同時寫進去打架。
- **資料庫 = Google Sheet**：每個功能對應一個分頁（tab），第一列是欄位名稱，下面就是資料。

前端和後端之間只靠一條 GAS 網址溝通。這條網址和登入密碼存在瀏覽器的 localStorage，第一次開網頁時會問你。

## 各分頁（資料表）

- `Expenses_DB`（日常記帳）：日期、類別、項目、金額、帳戶、收支
- `Asset_Log`（資產與現金流水帳）：日期、帳戶、資產類別、項目名稱、數量、金額、交易型態
- `Portfolio`（持股現況與即時報價）：項目名稱、持有數量、即時單價、昨日收盤、總市值、個股 Beta、加權 Beta
- `History_Log`（資產快照存檔）：日期、資產名稱、持有單位、現淨值、總成本
- `Habit_Log`（習慣打卡）：日期、英文、運動習慣、讀書習慣、睡覺習慣
- `Events_DB`（倒數事件）：事件名稱、日期、類型（`fixed` 一次性 / `yearly` 每年重複）
- `Wishlist_DB`（夢想購物車）：日期、項目名稱、預估金額、連結/備註
- `Media_Log`（知識書櫃）：日期、類別、標題、評分、心得
- `Settings`（選用，調目標和預算用）：項目、數值

後端寫入是「對欄位名稱」而不是「對欄位順序」，所以就算我哪天在試算表中間插一欄、或把順序調動，資料還是會落在正確的欄位。`Asset_Log` 比較特別：它前面有一欄雜訊、結尾有個空欄，而且型態欄是叫「交易型態」不是「型態」—— 這些後端都有對應好，動它之前要記得。

## 功能簡述

首頁是年度進度條加自訂倒數事件，下面是每天的習慣打卡，以及一條把「有沒有打卡」換算成每天乘 1.01 或 0.99 的複利成長曲線。

記帳頁上面是「生活可用餘額」「本月已花 / 預算 / 儲蓄率」，中間是記帳表單（快速帶入的按鈕會依我最常記的項目自動排序、並帶上次的金額），還有轉帳、本月對上月的支出比較、可搜尋篩選的最近紀錄。

資產頁算總資產淨值、第一桶金目標進度和預估達標時間，以及成本、損益、ROI、組合 Beta；下面有資產趨勢圖、配置圓餅、持股明細、每月收入（分主動和被動）。股票交易的表單會自動算手續費和證交稅，買或賣會一次寫進兩筆（扣款/入帳加上庫存增減），用後端批次寫入確保兩筆要嘛都成功、要嘛都不寫。

更多頁是各帳戶餘額對帳、夢想購物車（會自動判斷現在的錢夠不夠買）、還有閱讀觀影的紀錄。

## 黃金價格

`Portfolio` 裡股票的即時價多半是用 Google Sheet 的公式（GOOGLEFINANCE 之類）自己長出來的。只有黃金沒有現成公式，所以我在 `backend.gs` 裡寫了兩個自訂函數：

- `=GOLD_NOW()`：抓台灣銀行黃金存摺的「本行買進」現價（新台幣 / 公克）
- `=GOLD_PREV()`：抓前一個營業日的價

在 Google Sheet 的儲存格直接打 `=GOLD_NOW()` 就會去台銀牌價頁抓。想改成「本行賣出」價的話，把函數裡的 `.buy` 改成 `.sell` 即可。

注意：這兩個是寫在後端那支 `.gs` 裡的自訂函數。之前我把整份 `backend.gs` 覆蓋掉時，不小心連它們一起刪了，結果黃金那一列全變成 `#NAME?`。所以以後更新後端，記得不要把這兩個函數弄不見。

## 目標金額、每月預算

第一桶金目標和每月預算這兩個數字，可以用一個叫 `Settings` 的分頁來改，不必動程式：

| 項目 | 數值 |
| --- | --- |
| 第一桶金目標 | 1500000 |
| 每月預算 | 15000 |

A 欄的字只要「包含」關鍵字就抓得到（含「目標」或「桶金」當成目標，含「預算」當成月預算）。沒有這個分頁的話，程式會用內建預設值（目標 150 萬、預算 15000）。建好之後要改金額，直接改試算表裡的數字、重新整理網頁就好。

## 從零架起來

1. 開一份 Google Sheet，照上面的格式把分頁和欄位建好。
2. 擴充功能 → Apps Script，把 `backend.gs` 整份貼進去、存檔。
3. 部署 → 新增部署 → 類型選「網頁應用程式」→ 執行身分設成你自己、誰可以存取選「任何人」→ 部署，複製拿到的網址（結尾是 `/exec`）。
4. 把前端放上 GitHub Pages（這個 repo 本身就是）。
5. 開網頁，貼上剛剛那條 GAS 網址、設一組密碼。完成。

## 改版怎麼更新（這裡最容易踩雷）

前端和後端是分開更新的，要分清楚：

- **改前端（`index.html` 那些）**：push 到這個 repo，GitHub Pages 過一兩分鐘自動更新，重新整理網頁就生效。
- **改後端（`backend.gs`）**：push 到 GitHub 不會生效。真正在跑的是 GAS 雲端那一份，要自己把程式碼貼回 Apps Script，然後「部署 → 管理部署 → 編輯現有那個 → 版本選『新版本』→ 部署」。一定要用「編輯現有」，這樣網址不會變、前端什麼都不用改。如果按到「新增部署」會生出一條新網址，那就得回前端重設網址。

## PWA（手機桌面 App）

有 `manifest.json` 和 `service-worker.js`，可以把網頁「加入主畫面」變成像一支獨立 App（全螢幕、有自己的圖示）。iPhone 要用 Safari 開才有「加入主畫面」的選項。

離線的部分：service worker 會把 App 本體和「上次抓到的資料」快取起來，所以沒網路時打開不會一片空白，看得到上次的數字。但它不是真的離線運作 —— 看到的是上次連線時的快照，不是即時資料，而且離線不能記帳（寫入一定要連網）。要更新離線快取就是連網重開一次。改大版時把 `service-worker.js` 裡的版本號 `life-dashboard-v1` 往上加，舊快取才會被換掉。

## 已知的限制（老實講）

- **登入密碼不是真的資安**。它只擋網頁畫面而已。真正的鑰匙是那條 GAS 網址，誰知道網址就能直接讀寫整份試算表，後端目前沒做驗證。所以網址別外流；`config.js`（裡面有網址和密碼）也用 `.gitignore` 擋掉了，不要上傳。（補充：`index.html` 現在是用 localStorage 問你網址，`config.js` 其實是早期留下的、目前沒在載入，但裡面是機密所以照樣別上傳。）
- **刪除只比對最近 30 筆**，而且是用「日期＋項目＋金額」去湊。所以太舊的刪不掉，同一天兩筆金額一樣的也可能刪錯。要根治得幫每筆加一個唯一 ID，還沒做。
- **沒有編輯功能**，要改一筆只能刪掉重記。
- **資產趨勢圖靠手動快照**。資產頁有個「快照」鈕，按了才會把當下資產存進 `History_Log` 當一個時間點。沒定期按，趨勢圖就沒東西可畫。
- **股價和金價是試算表公式**，公式壞掉（例如金價函數失效變 `#NAME?`）那格就沒值。前端有做防呆，抓不到的數字一律當 0，至少不會把整個總資產算成 NaN。

## 檔案

- `index.html` — 前端整包（畫面、樣式、邏輯都在裡面）
- `backend.gs` — GAS 後端（`doGet` / `doPost` 加上 GOLD 函數），要貼到 Apps Script
- `manifest.json`、`service-worker.js` — PWA 用
- `config.example.js` — 設定檔範本（真正的 `config.js` 不進版控）
